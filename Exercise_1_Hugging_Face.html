Build a sentiment analysis pipeline manually

In this exercise, you use Hugging Face Transformers to understand what happens behind the scenes when you run a sentiment-analysis pipeline.

Tasks performed in this exercise:

  • Create a Python project in Visual Studio Code.
  • Install PyTorch and Hugging Face Transformers.
  • Run a baseline sentiment-analysis pipeline.
  • Manually tokenize input text.
  • Pass tokenized inputs through a Transformer model.
  • Inspect hidden states, logits, and probabilities.
  • Compare the manual implementation with the high-level pipeline.

This exercise takes approximately 45–60 minutes to complete.

Important: The first execution downloads the model and tokenizer from Hugging Face. You need an internet connection the first time you run the scripts.

Scenario

You work with a customer feedback team that receives short comments from users. The team wants to classify comments as positive or negative, but before using an AI model in an application, you need to understand the complete inference flow.

Instead of only using the easy pipeline() function, you will manually rebuild the same process step by step:

  1. Convert raw text into tokens.
  2. Convert tokens into numerical input IDs.
  3. Create attention masks.
  4. Send tensors into a pretrained Transformer model.
  5. Read the model's raw logits.
  6. Convert logits into probabilities.
  7. Select the final sentiment label.

In this section, you load the base Transformer model with AutoModel. This returns hidden states instead of final sentiment labels.

    "I've been waiting for a Hugging Face course my whole life.",
    "I hate this so much!"
]

    texts,
    padding=True,
    truncation=True,
    return_tensors="pt"

outputs = model(**inputs)

print("Output object:")
print(type(outputs))

print("\nLast hidden state shape:")

  1. Run the script.
python 04_base_model_hidden_states.py

The shape usually follows this pattern:

batch size x sequence length x hidden size

For example, torch.Size([2, 16, 768]) means two sentences, sixteen token positions, and a 768-value vector for each token position.

Convert logits into probabilities

In this section, you apply softmax to convert raw logits into probabilities.

  1. Create a file named 06_logits_to_probabilities.py.
  2. Add the following code to the file.
import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification

checkpoint = "distilbert-base-uncased-finetuned-sst-2-english"

tokenizer = AutoTokenizer.from_pretrained(checkpoint)
model = AutoModelForSequenceClassification.from_pretrained(checkpoint)

texts = [
    "I've been waiting for a Hugging Face course my whole life.",
    "I hate this so much!"
]

inputs = tokenizer(
    texts,
    padding=True,
    truncation=True,
    return_tensors="pt"
)

outputs = model(**inputs)
probabilities = torch.nn.functional.softmax(outputs.logits, dim=-1)

print("Raw logits:")
print(outputs.logits)

print("\nProbabilities:")
print(probabilities)

print("\nLabel mapping:")
print(model.config.id2label)

print("\nFinal predictions:")
for index, text in enumerate(texts):
    negative_score = probabilities[index][0].item()
    positive_score = probabilities[index][1].item()

    print("Text:", text)
    print("NEGATIVE score:", negative_score)
    print("POSITIVE score:", positive_score)

    if positive_score > negative_score:
        print("Final label: POSITIVE")
    else:
        print("Final label: NEGATIVE")

    print("-" * 80)
  1. Run the script.
python 06_logits_to_probabilities.py

Softmax converts raw model scores into probability-like values between 0 and 1.

Build the full manual pipeline

In this section, you combine tokenization, model inference, softmax, and label selection into one complete script.

  1. Create a file named 07_manual_pipeline.py.
  2. Add the following code to the file.
import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification

checkpoint = "distilbert-base-uncased-finetuned-sst-2-english"

texts = [
    "I've been waiting for a Hugging Face course my whole life.",
    "I hate this so much!",
    "The product is okay, but delivery was very slow.",
    "This is one of the best learning experiences I have had."
]

print("Loading tokenizer...")
tokenizer = AutoTokenizer.from_pretrained(checkpoint)

print("Loading model...")
model = AutoModelForSequenceClassification.from_pretrained(checkpoint)

print("\nStep 1: Tokenizing text")
inputs = tokenizer(
    texts,
    padding=True,
    truncation=True,
    return_tensors="pt"
)

print("Input IDs shape:", inputs["input_ids"].shape)
print("Attention mask shape:", inputs["attention_mask"].shape)

print("\nStep 2: Running model")
outputs = model(**inputs)

print("Logits shape:", outputs.logits.shape)
print("Logits:")
print(outputs.logits)

print("\nStep 3: Applying softmax")
probabilities = torch.nn.functional.softmax(outputs.logits, dim=-1)

print("Probabilities:")
print(probabilities)

print("\nStep 4: Converting probabilities to labels")
label_mapping = model.config.id2label

for index, text in enumerate(texts):
    predicted_class_id = torch.argmax(probabilities[index]).item()
    predicted_label = label_mapping[predicted_class_id]
    predicted_score = probabilities[index][predicted_class_id].item()

    print("\nText:", text)
    print("Predicted label:", predicted_label)
    print("Confidence score:", round(predicted_score, 4))
  1. Run the script.
python 07_manual_pipeline.py

Compare the manual pipeline with the Hugging Face pipeline

In this section, you verify that your manual process gives results similar to the high-level pipeline.

  1. Create a file named 08_compare_pipeline_vs_manual.py.
  2. Add the following code to the file.
import torch
from transformers import pipeline, AutoTokenizer, AutoModelForSequenceClassification

checkpoint = "distilbert-base-uncased-finetuned-sst-2-english"

texts = [
    "I've been waiting for a Hugging Face course my whole life.",
    "I hate this so much!",
    "The product is okay, but delivery was very slow.",
    "This is one of the best learning experiences I have had."
]

print("Running Hugging Face pipeline...")
classifier = pipeline("sentiment-analysis", model=checkpoint)
pipeline_results = classifier(texts)

print("\nRunning manual pipeline...")
tokenizer = AutoTokenizer.from_pretrained(checkpoint)
model = AutoModelForSequenceClassification.from_pretrained(checkpoint)

inputs = tokenizer(
    texts,
    padding=True,
    truncation=True,
    return_tensors="pt"
)

outputs = model(**inputs)
probabilities = torch.nn.functional.softmax(outputs.logits, dim=-1)

manual_results = []

for index in range(len(texts)):
    predicted_class_id = torch.argmax(probabilities[index]).item()
    predicted_label = model.config.id2label[predicted_class_id]
    predicted_score = probabilities[index][predicted_class_id].item()

    manual_results.append({
        "label": predicted_label,
        "score": predicted_score
    })

print("\nComparison:")
for text, pipeline_result, manual_result in zip(texts, pipeline_results, manual_results):
    print("Text:", text)
    print("Pipeline result:", pipeline_result)
    print("Manual result:", manual_result)
    print("-" * 80)
  1. Run the script.
python 08_compare_pipeline_vs_manual.py

The labels and scores should be very close. This confirms that the high-level pipeline and your manual implementation are performing the same core process.

Student challenge

In this section, you modify the inputs and test your own sentences.

  1. Create a file named 09_student_challenge.py.
  2. Add the following code to the file.
import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification

checkpoint = "distilbert-base-uncased-finetuned-sst-2-english"

texts = [
    "The training session was very useful and easy to understand.",
    "The application failed again and the support experience was terrible.",
    "The service was average, nothing special.",
    "I am happy with the final result, but the process was slow."
]

tokenizer = AutoTokenizer.from_pretrained(checkpoint)
model = AutoModelForSequenceClassification.from_pretrained(checkpoint)

inputs = tokenizer(
    texts,
    padding=True,
    truncation=True,
    return_tensors="pt"
)

outputs = model(**inputs)
probabilities = torch.nn.functional.softmax(outputs.logits, dim=-1)

for index, text in enumerate(texts):
    negative_probability = probabilities[index][0].item()
    positive_probability = probabilities[index][1].item()

    if positive_probability > negative_probability:
        label = "POSITIVE"
        score = positive_probability
    else:
        label = "NEGATIVE"
        score = negative_probability

    print("Text:", text)
    print("Negative probability:", round(negative_probability, 4))
    print("Positive probability:", round(positive_probability, 4))
    print("Final label:", label)
    print("Confidence:", round(score, 4))
    print("-" * 80)
  1. Run the script.
python 09_student_challenge.py

  2. Run the script again and compare the predictions.

Review what you learned

In this exercise, you learned that a Hugging Face pipeline combines three main operations:

  1. Preprocessing: Tokenization converts raw text into model-ready tensors.
  2. Model inference: The Transformer model processes the tensors and produces outputs.
  3. Postprocessing: Logits are converted into probabilities and final labels.

The complete flow is:

Raw text
   ↓
Tokenizer
   ↓
Input IDs + Attention Mask
   ↓
Transformer model
   ↓
Classification logits
   ↓
Softmax probabilities
   ↓
Final sentiment label

Knowledge check

Question 1

What are the three major stages hidden inside a Hugging Face pipeline?

Answer: Preprocessing, model inference, and postprocessing.

Question 2

Why does a Transformer model need a tokenizer?

Answer: Because Transformer models cannot directly process raw text. The tokenizer converts text into numbers.

Question 3

What are input_ids?

Answer: They are integer IDs representing the tokens in the input text.

Question 4

What does the attention_mask do?

Answer: It tells the model which positions are real tokens and which positions are padding.

Question 5

What is the difference between AutoModel and AutoModelForSequenceClassification?

Answer: AutoModel returns hidden states. AutoModelForSequenceClassification adds a classification head and returns logits for labels.

Question 6

What are logits?

Answer: Logits are raw, unnormalized model scores before conversion into probabilities.

Question 7

Why do we apply softmax?

Answer: Softmax converts logits into probability-like values that can be used to choose the final label.
