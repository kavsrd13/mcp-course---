Explore Transformer pipelines in Google Colab

Use Hugging Face Transformers to solve common text, image, and audio tasks with the high-level pipeline() API.

Beginner60-75 minutesGoogle Colab
Based on: Hugging Face LLM Course, Transformers, what can they do?. This exercise reorganizes the concepts into a guided Colab lab.

Learning objectives

After completing this exercise, you will be able to:

  • Explain how a pipeline combines preprocessing, model inference, and postprocessing.
  • Run sentiment analysis, zero-shot classification, text generation, mask filling, NER, summarization, and translation.
  • Select a specific model from the Hugging Face Hub.
  • Try image classification and automatic speech recognition as optional multimodal tasks.

Prerequisites

  • A Google account that can open Google Colab.
  • Basic Python familiarity: variables, lists, dictionaries, and function calls.
  • Internet access. The notebook downloads pretrained models from the Hugging Face Hub.
Download notice: The first run of each task may take several minutes because Colab must download a model. Later calls reuse the model cached in the current Colab session.

3. Import pipeline and select the device

Colab cell 2 - Imports and device
import torch
from transformers import pipeline

device = 0 if torch.cuda.is_available() else -1
print("Using:", "GPU" if device == 0 else "CPU")
How a pipeline works: It converts input into model-ready values, runs the model, and converts raw predictions into readable Python dictionaries or text.

4. Sentiment analysis

Sentiment analysis classifies text as positive or negative and returns a confidence score.

Colab cell 3 - Sentiment analysis
sentiment = pipeline(
    "sentiment-analysis",
    model="distilbert/distilbert-base-uncased-finetuned-sst-2-english",
    device=device,
)

sentiment("I enjoyed learning how Transformer pipelines work.")

The result should be a list containing a label such as POSITIVE and a score between 0 and 1.

Pass multiple sentences in one call:

Colab cell 4 - Batch input
sentiment([
    "The explanations were clear and helpful.",
    "I found this lesson frustrating and confusing.",
])

Try it Replace both sentences with feedback from your own class or project.

5. Zero-shot classification

Zero-shot classification lets you supply labels at inference time instead of training a classifier for those exact categories.

Colab cell 5 - Zero-shot classification
zero_shot = pipeline(
    "zero-shot-classification",
    model="facebook/bart-large-mnli",
    device=device,
)

zero_shot(
    "This lesson explains how to use pretrained language models.",
    candidate_labels=["education", "business", "sports", "politics"],
)

Inspect the order of labels and their scores. The highest-scoring label should be related to education.

Teaching point: “Zero-shot” means the model was not fine-tuned using your exact list of labels. The labels are part of the request.

6. Text generation with a Hub model

Create a generation pipeline using the compact HuggingFaceTB/SmolLM2-360M model.

Colab cell 6 - Text generation
generator = pipeline(
    "text-generation",
    model="HuggingFaceTB/SmolLM2-360M",
    device=device,
)

generator(
    "In this course, students will learn how to",
    max_new_tokens=30,
    num_return_sequences=2,
    do_sample=True,
    temperature=0.8,
)

You should receive two completions. Their wording may differ each time because sampling introduces randomness.

Try it Change temperature to 0.2, rerun the cell, and compare how conservative the output becomes.

7. Fill a masked word

A fill-mask model predicts likely words for a special mask token.

Colab cell 7 - Mask filling
unmasker = pipeline(
    "fill-mask",
    model="FacebookAI/roberta-base",
    device=device,
)

unmasker(
    "Transformer models can solve many <mask> tasks.",
    top_k=3,
)

The result contains three candidate sequences, token strings, and scores.

Model-specific token: RoBERTa uses <mask>. Other model families may use a different mask token, so check the model card or tokenizer.

8. Named entity recognition

Named entity recognition identifies spans representing people, organizations, locations, and other entity types.

Colab cell 8 - Named entity recognition
ner = pipeline(
    "ner",
    model="dslim/bert-base-NER",
    aggregation_strategy="simple",
    device=device,
)

ner("Asha works for Microsoft in Bengaluru and visits London in July.")

Look for entity groups such as PER, ORG, and LOC. The aggregation strategy combines subword pieces into readable entities.

9. Summarization

Summarization creates a shorter version while retaining important information.

Colab cell 9 - Summarization
summarizer = pipeline(
    "summarization",
    model="sshleifer/distilbart-cnn-12-6",
    device=device,
)

article = """
Transformer models are used for many natural language processing tasks.
The Hugging Face pipeline API combines a tokenizer, a pretrained model,
and task-specific postprocessing behind a simple function call. This makes
it convenient for experimentation and teaching. For production systems,
developers still need to evaluate model quality, latency, cost, safety,
and whether the selected model is appropriate for the intended data.
"""

summarizer(article, max_length=55, min_length=20, do_sample=False)

The result contains summary_text. Compare it with the original and identify which details were removed.

11. Optional: image classification

This step demonstrates that the pipeline API also supports images.

Colab cell 11 - Image classification
image_classifier = pipeline(
    "image-classification",
    model="google/vit-base-patch16-224",
    device=device,
)

image_url = (
    "https://huggingface.co/datasets/huggingface/documentation-images/"
    "resolve/main/pipeline-cat-chonk.jpeg"
)

image_classifier(image_url)[:5]

The output is a ranked list of image labels and scores. Predictions are model estimates and may not match the everyday label you expected.

12. Optional: automatic speech recognition

Use a compact Whisper checkpoint to transcribe a short audio file.

Colab cell 12 - Speech recognition
transcriber = pipeline(
    "automatic-speech-recognition",
    model="openai/whisper-tiny.en",

)

audio_url = "https://huggingface.co/datasets/Narsil/asr_dummy/resolve/main/mlk.flac"

The result should be a dictionary containing transcribed text.

Troubleshooting

ProblemSuggested action
Colab disconnects or forgets variablesReconnect and use Runtime > Run all. Colab sessions are temporary.
CUDA out-of-memory errorRestart the runtime and run fewer models at once, or switch to CPU. Deleting a Python variable alone may not immediately release all GPU memory.
Model download is slowWait for the first download to finish. Do not repeatedly rerun the same loading cell.
Output differs from the exampleGeneration is nondeterministic and model versions can change. Check the output structure and task behavior rather than exact wording.
A model requires permissionUse a public alternative. This lab does not require a Hugging Face access token.
Translation tokenizer errorRerun the package installation cell and confirm sentencepiece and sacremoses installed successfully.
