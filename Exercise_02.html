Behind a Transformers sentiment pipeline

Open a prepared Google Colab notebook and manually reproduce tokenization, model inference, and prediction postprocessing.

Beginner-Intermediate45-60 minutesGoogle Colab + PyTorch
Based on: Hugging Face LLM Course, Behind the pipeline.

Learning objectives

  • Explain why Transformer models require numeric tensors instead of raw text.
  • Inspect tokens, token IDs, special tokens, padding, and attention masks.
  • Interpret the dimensions of a hidden-state tensor.
  • Distinguish a base Transformer from a task-specific model head.
  • Convert logits into probabilities and labels.
  • Verify that manual processing reproduces pipeline().

The complete flow

Transformer
Logits
Labels

The high-level pipeline performs all these stages. In this exercise, you pause after each one and inspect the intermediate values.

2. Install and import the libraries

Run the package cell. Colab already includes PyTorch, and this installs the Transformers library:

Notebook cell - Install

Run the imports and define the checkpoint and two example sentences:

Notebook cell - Setup
import torch
from transformers import (
    AutoModel,
    AutoModelForSequenceClassification,
    AutoTokenizer,
    pipeline,
)

checkpoint = "distilbert/distilbert-base-uncased-finetuned-sst-2-english"
texts = [
    "I've been waiting for a Hugging Face course my whole life.",
    "I dislike this lesson very much!",
]

print("PyTorch version:", torch.__version__)
print("Number of texts:", len(texts))
Checkpoint consistency: The tokenizer and every model must use the same checkpoint because preprocessing must match the way the model was trained.

3. Inspect tokens and token IDs

Run the tokenizer inspection cell:

Notebook cell - Tokens and IDs
tokenizer = AutoTokenizer.from_pretrained(checkpoint)

for text in texts:
    tokens = tokenizer.tokenize(text)
    token_ids = tokenizer.convert_tokens_to_ids(tokens)
    print("TEXT:", text)
    print("TOKENS:", tokens)
    print("TOKEN IDS:", token_ids)
    print()
  1. Find words that were divided into subword tokens. DistilBERT marks continuation pieces with ##.
  2. Notice that punctuation can be its own token.
  3. Confirm that every token maps to an integer in the tokenizer vocabulary.
Discussion: Tokenization is not simply splitting on spaces. Subwords let a fixed vocabulary represent unfamiliar or uncommon words.

4. Create model-ready tensors

Notebook cell - Batch encoding
inputs = tokenizer(
    texts,
    padding=True,
    truncation=True,
    return_tensors="pt",
)

print(inputs)
print("input_ids shape:", inputs["input_ids"].shape)
print("attention_mask shape:", inputs["attention_mask"].shape)

The tokenizer now performs several extra operations:

  • Adds special tokens needed by the model.
  • Pads the shorter sentence so both rows have the same length.
  • Creates PyTorch tensors because return_tensors="pt".
  • Creates an attention mask to distinguish real tokens from padding.

Inspect the batch token by token

Notebook cell - Token table
for row in range(len(texts)):
    ids = inputs["input_ids"][row]
    mask = inputs["attention_mask"][row]
    tokens = tokenizer.convert_ids_to_tokens(ids)

    print(f"SEQUENCE {row + 1}")
    for token, token_id, attention in zip(tokens, ids.tolist(), mask.tolist()):
        print(f"{token:15} id={token_id:5} attention={attention}")
    print()

Find the [CLS], [SEP], and [PAD] tokens. Padding positions should have attention value 0; real and special tokens should have value 1.

5. Run only the base Transformer

Notebook cell - Hidden states
base_model = AutoModel.from_pretrained(checkpoint)
base_model.eval()

with torch.no_grad():
    base_outputs = base_model(**inputs)

hidden_states = base_outputs.last_hidden_state
print("Hidden-state shape:", hidden_states.shape)
print("Batch size:", hidden_states.shape[0])
print("Sequence length:", hidden_states.shape[1])
print("Hidden size:", hidden_states.shape[2])
print("First token, first 10 values:", hidden_states[0, 0, :10])

Interpret the three dimensions:

DimensionMeaning
Batch sizeThe number of input sequences processed together.
Sequence lengthThe padded number of tokens in each sequence.
Hidden sizeThe contextual vector size produced for each token, 768 for this model.
Important: Hidden states are learned numeric features. They are not yet positive or negative probabilities.

6. Add the sentiment classification head

Notebook cell - Logits
classifier_model = AutoModelForSequenceClassification.from_pretrained(checkpoint)
classifier_model.eval()

with torch.no_grad():
    outputs = classifier_model(**inputs)

logits = outputs.logits
print("Logits:")
print(logits)
print("Logits shape:", logits.shape)
print("Label mapping:", classifier_model.config.id2label)

The output shape is [2, 2]: two input sentences and two possible labels. The task head projects the Transformer representation down to two raw scores.

Do not interpret logits as probabilities: logits may be negative and do not need to sum to 1.

7. Convert logits to predictions

Notebook cell - Softmax and labels
probabilities = torch.nn.functional.softmax(logits, dim=-1)
predicted_class_ids = probabilities.argmax(dim=-1)

manual_results = []
for text, class_id, scores in zip(texts, predicted_class_ids, probabilities):
    class_id = class_id.item()
    result = {
        "text": text,
        "label": classifier_model.config.id2label[class_id],
        "score": scores[class_id].item(),
    }
    manual_results.append(result)

print("Probabilities:")
print(probabilities)
print("Row sums:", probabilities.sum(dim=-1))
print("Manual results:")
for result in manual_results:
    print(result)
  1. Confirm that every probability is between 0 and 1.
  2. Confirm that each row sum is approximately 1.
  3. Observe how argmax selects the largest class probability.
  4. Observe how id2label maps class 0 or 1 to NEGATIVE or POSITIVE.

8. Compare the manual process with pipeline()

Notebook cell - Pipeline comparison
sentiment_pipeline = pipeline(
    "sentiment-analysis",
    model=checkpoint,
    tokenizer=checkpoint,
)

pipeline_results = sentiment_pipeline(texts)

for manual, automatic in zip(manual_results, pipeline_results):
    print("Manual:  ", manual["label"], round(manual["score"], 6))
    print("Pipeline:", automatic["label"], round(automatic["score"], 6))
    print()

assert [item["label"] for item in manual_results] == [
    item["label"] for item in pipeline_results
]
print("The manual and pipeline labels match.")
Expected result: The labels and scores should match, apart from tiny floating-point rounding differences. You have reproduced the pipeline's major stages.

Troubleshooting

ProblemSuggested action
Variables are undefinedRun the notebook from the top. Later cells depend on earlier tokenizer and model variables.
Download takes a long timeWait for the first model download to complete. The checkpoint is cached for the current runtime.
Tensor shape differsSequence length changes when you edit the text because padding uses the longest sequence in the batch.
Manual and pipeline scores differUse the same checkpoint and tokenizer, call eval(), and keep torch.no_grad() around inference.
GPU memory errorRestart the runtime or use CPU. This small notebook does not require a GPU.
Summary
A sentiment pipeline is a convenient wrapper around preprocessing, Transformer inference, a classification head, and postprocessing: text -> tokens -> IDs and masks -> hidden states -> logits -> softmax -> labels.
