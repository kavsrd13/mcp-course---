Use a remote MCP server with Azure OpenAI

Create a Microsoft Foundry resource, deploy a model, expose a local MCP server through a Dev Tunnel, and let an Azure-hosted model call the server.

Intermediate60-75 minutesAzure + Python
Cost notice: This exercise creates billable Azure resources. Model availability and quotas vary by region and subscription. Complete the cleanup section when finished.

Learning objectives

  • Create a Microsoft Foundry project and model deployment.
  • Use the Azure OpenAI v1 endpoint with the OpenAI Python SDK.
  • Run an MCP server with Streamable HTTP and expose it over HTTPS.
  • Attach the remote server as an MCP tool in the Responses API.

Prerequisites

  • An Azure subscription with permission to create a resource group, Foundry resource, project, and model deployment.
  • Python 3.10 or later and Visual Studio Code.
  • Windows Package Manager (winget) for installing Microsoft Dev Tunnels.

1. Create a Microsoft Foundry project

  1. Open https://ai.azure.com and sign in with the account connected to your Azure subscription.
  2. Select Create new project. If the portal first asks you to choose or create a Foundry resource, choose Create a new resource.
  3. Use values similar to the following. Names must be unique where the portal requires it.
SettingSuggested classroom value
Project namemcp-classroom-project
Foundry resource namemcp-classroom-<your-initials>
SubscriptionYour available Azure subscription
Resource groupCreate rg-mcp-classroom
RegionA region that offers the model and has quota for your subscription
  1. Select Create and wait for the project to open.
  2. Keep the project page open. Portal labels can differ slightly between Foundry updates; use Build, Models, or Models + endpoints when following the next section.

2. Deploy a GPT-4.1 model

  1. In the Foundry project, open Models or Models + endpoints.
  2. Select Deploy model and then Deploy base model.
  3. Search for gpt-4.1, select it, and choose Confirm.
  4. Set the deployment name to gpt-4.1. For a classroom exercise, accept the available standard deployment type and default capacity.
  5. Select Deploy and wait until provisioning succeeds.
If GPT-4.1 is unavailable: choose another Responses API model that supports remote MCP tools in your region. Record the exact deployment name and use it later for AZURE_OPENAI_DEPLOYMENT.
Important: Azure API calls use the deployment name in the model field, even when the deployment name and catalog model name happen to be identical.

3. Find the endpoint and key

  1. Open the Foundry resource in the Azure portal. You can usually reach it from the project overview or resource details.
  2. Under Resource Management, open Keys and Endpoint.
  3. Copy one key and the Azure OpenAI-compatible endpoint. Do not paste the key into a source file.
  4. Build the base URL by appending /openai/v1/ to the resource endpoint.

5. Create the remote MCP server

Create server.py and paste this code:

server.py
from datetime import datetime, timezone

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("classroom-remote-mcp", host="0.0.0.0", port=8000)


@mcp.tool()
def get_course(topic: str) -> dict:
    """Return a tiny course recommendation for a topic."""
    return {
        "topic": topic,
        "title": f"Introduction to {topic}",
        "duration_minutes": 20,
        "generated_at": datetime.now(timezone.utc).isoformat(),
    }


if __name__ == "__main__":
    mcp.run(transport="streamable-http")

This server listens on port 8000 and exposes its Streamable HTTP endpoint at /mcp.

6. Create the Azure OpenAI agent script

Create agent.py and paste:

agent.py
import os

from openai import OpenAI

client = OpenAI(
    api_key=os.environ["AZURE_OPENAI_API_KEY"],
    base_url=os.environ["AZURE_OPENAI_BASE_URL"],
)

response = client.responses.create(
    model=os.environ["AZURE_OPENAI_DEPLOYMENT"],
    input="Suggest a short course for learning MCP. Use the course server.",
    tools=[
        {
            "type": "mcp",
            "server_label": "classroom_courses",
            "server_url": os.environ["MCP_SERVER_URL"],
            "require_approval": "never",
        }
    ],
)

print(response.output_text)
Data flow: Your script calls Azure OpenAI. Azure connects to the public MCP URL, discovers get_course, lets the model call it, and returns the model's final answer.

7. Start the MCP server

In the first VS Code terminal, run:

Terminal 1 - PowerShell
.\.venv\Scripts\Activate.ps1
python server.py

Leave this terminal running. The local endpoint is http://localhost:8000/mcp.

8. Expose the server with Microsoft Dev Tunnel

Azure cannot connect to your laptop's localhost. Open a second terminal and install Dev Tunnels if needed:

Terminal 2 - PowerShell
winget install Microsoft.devtunnel
devtunnel user login

Restart the terminal if devtunnel is not found after installation. Then host a temporary public tunnel:

Terminal 2 - PowerShell
devtunnel host -p 8000 --allow-anonymous

Copy the HTTPS URL shown by the command, for example:

Example output
https://abc123-8000.region.devtunnels.ms

Your MCP server URL is that value plus /mcp:

MCP_SERVER_URL example
https://abc123-8000.region.devtunnels.ms/mcp
Classroom-only setting: --allow-anonymous makes the tunnel reachable without signing in. Do not expose private data or production tools. Press Ctrl+C as soon as the exercise is complete.

9. Configure credentials in VS Code

Open a third terminal. Set the four environment variables using your own values:

Terminal 3 - PowerShell
.\.venv\Scripts\Activate.ps1
$env:AZURE_OPENAI_API_KEY="PASTE-YOUR-KEY"
$env:AZURE_OPENAI_BASE_URL="https://YOUR-RESOURCE.services.ai.azure.com/openai/v1/"
$env:AZURE_OPENAI_DEPLOYMENT="gpt-4.1"
$env:MCP_SERVER_URL="https://YOUR-DEV-TUNNEL-URL/mcp"
Check carefully: The base URL ends in /openai/v1/. The MCP URL ends in /mcp. The deployment variable is the deployment name, not necessarily the catalog model name.

10. Run the agent

Terminal 3 - PowerShell
python agent.py
Expected output: A short recommendation such as “Introduction to MCP,” including the 20-minute duration returned by get_course.

Watch Terminal 1 while the request runs. The server should receive initialization, discovery, and tool-call traffic from the Azure service through the tunnel.

Troubleshooting

SymptomLikely fix
401 or authentication errorCopy the key again and confirm it belongs to the same resource used in the base URL.
404 from Azure OpenAIConfirm the base URL contains /openai/v1/ and the deployment name is exact.
Model deployment not foundUse the deployment name shown under Models + endpoints, not just the model family name.
Azure cannot reach MCP serverKeep both server and Dev Tunnel terminals running; verify the public URL ends in /mcp.
MCP tool unsupportedUse a model/region that supports remote MCP tools with the Responses API.
Environment variable missingSet variables in the same terminal where you run python agent.py.

Cleanup

  1. Press Ctrl+C in the agent server and Dev Tunnel terminals.
  2. In the Azure portal, open resource group rg-mcp-classroom.
  3. Select Delete resource group, enter its name to confirm, and complete deletion.
  4. Run deactivate in local Python terminals.
Summary
You connected an Azure-hosted model to a local MCP server through a public HTTPS endpoint and used the OpenAI Responses API to let the model discover and invoke an MCP tool.
