Build a RAG Agent with Azure AI Foundry and Azure AI Search

In this exercise, you build a Retrieval Augmented Generation (RAG) solution by combining Azure AI Foundry model deployments with Azure AI Search.

Tasks performed in this exercise:

  • Clone the starter repository and open it in Visual Studio Code.
  • Create Azure resources from Azure portal (no Azure CLI provisioning).
  • Deploy a chat model and an embedding model in Azure AI Foundry.
  • Fill environment settings required by the app.
  • Complete critical RAG code sections in the Python files.
  • Run ingestion and chat to verify grounded responses.
This exercise takes approximately 50 minutes.

Before you start

  • Azure subscription with resource creation rights.
  • Visual Studio Code and Python 3.10+.
  • Access to Azure portal at portal.azure.com.
Sign in to Azure portal with the subscription you will use for this lab. Keep the same account open for all steps.

Download project files

Clone the repository and open the project in Visual Studio Code.

git clone https://github.com/kavsrd13/RAG-Project.git
cd RAG-Project
code .
Use the RAG_Agent_Framework folder from the repository root for all remaining tasks.

Create Azure resources manually

Create each service from Azure portal.

  1. Open Resource groups and create a group named <your-resource-group> in your chosen region.
  2. Create an Azure AI services resource in that group (SKU S0).
  3. Create an Azure AI Search resource in the same group (SKU Basic).
  4. Open Azure AI Foundry at ai.azure.com and create a project.
  5. Connect the project to your Azure AI services resource.

Deploy model endpoints

In Azure AI Foundry project, deploy these models and keep deployment names exactly as entered.

Record connection settings

Collect values from portal and record them in your local .env file.

  1. Foundry endpoint: Azure AI Foundry project > Overview > Project endpoint
  2. Azure AI services key: Azure AI services resource > Keys and Endpoint
  3. Search endpoint and admin key: Azure AI Search resource > Keys

Run the RAG app

Run ingestion

python ingest.py

Run chat

python main.py

Ask two questions from the benefits documents and verify:

  • Retrieved chunks are printed first.
  • The answer references source document titles.
  • If context is missing, the app reports it does not have that information.
Expected output: grounded answer with source-specific chunk references in terminal output.

Troubleshooting

  • If authentication fails, re-check AZURE_OPENAI_API_KEY and AZURE_SEARCH_API_KEY values.
  • If model invocation fails, confirm deployment names match .env exactly.
  • If no results are returned, run ingest.py again and confirm AZURE_SEARCH_INDEX value.

Clean up resources

After completing the lab, delete the resource group from Azure portal to avoid additional charges.

  1. Open Resource groups in Azure portal.
  2. Select your lab resource group.
  3. Select Delete resource group and confirm the name.
