Test an MCP server with MCP Inspector

Use the protocol-focused Inspector to examine schemas, invoke capabilities, and observe how an MCP client communicates with a server.

Beginner30 minutesMCP Inspector

Learning objectives

  • Launch a stdio MCP server from Inspector.
  • Test valid and invalid tool input.
  • Read resources, retrieve prompts, and inspect protocol activity.

Prerequisites

  • Python 3.10 or later.
  • Node.js and npm.
  • Visual Studio Code with a PowerShell terminal.

1. Prepare the project

PowerShell
mkdir mcp-inspector-lab
cd mcp-inspector-lab
code .
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install "mcp[cli]"
Tip: If PowerShell blocks activation, run Set-ExecutionPolicy -Scope Process Bypass first.

2. Create a server designed for testing

Create server.py, paste the code, and save it.

server.py
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("inspector-lab")


@mcp.tool()
def grade(score: int, total: int = 100) -> dict:
    """Return a percentage and a simple pass/fail result."""
    if total <= 0:
        return {"error": "total must be greater than zero"}

    percentage = round(score / total * 100, 1)
    return {"percentage": percentage, "passed": percentage >= 40}


@mcp.resource("classroom://rules")
def classroom_rules() -> str:
    """Return sample classroom rules."""
    return "1. Ask questions\n2. Try the tool\n3. Explain what happened"


@mcp.prompt()
def quiz_question(subject: str, level: str = "beginner") -> str:
    """Create a prompt for generating one quiz question."""
    return (
        f"Create one {level} multiple-choice question about {subject}. "
        "Include four choices and identify the correct answer."
    )


if __name__ == "__main__":
    mcp.run()

3. Start Inspector

PowerShell
npx @modelcontextprotocol/inspector python server.py
  1. Open the URL shown in the terminal. Inspector normally opens it automatically.
  2. Use STDIO as the transport.
  3. Confirm command python and argument server.py, and select Connect.
  4. Observe the server name and the capabilities returned during initialization.
Concept: Before calling anything, an MCP client initializes the connection and asks what the server supports. Inspector turns those capability lists into tabs and forms.

4. Test the tool schema

  1. Open Tools and select grade.
  2. Notice that score is required and total has a default. These fields were generated from the Python function signature.
  3. Run each case below and compare the result.
ScoreTotalExpected result
75100{"percentage":75.0,"passed":true}
205040%, passed
200Error object from the function
Teaching point: Returning an error object is application behavior. Passing the wrong JSON type, such as text for score, is schema validation behavior and may be rejected before the function runs.

5. Test the resource and prompt

  1. Open Resources, select classroom://rules, and choose Read Resource.
  2. Confirm that the result is plain text containing three rules.
  3. Open Prompts and select quiz_question.
  4. Set subject to Model Context Protocol and level to beginner.
  5. Retrieve the prompt and inspect the generated message.

A resource supplies context. A prompt supplies reusable instructions. Neither one is automatically an action.

6. Observe protocol activity

  1. Repeat a tool call and watch Inspector's message or notification area.
  2. Identify requests such as capability listing and tool invocation.
  3. Disconnect and reconnect. Notice that capability discovery happens again for the new session.
Optional experiment: Rename grade to calculate_grade, save the file, reconnect, and confirm that the published tool name changes.

Troubleshooting

ProblemCheck
python not foundInstall Python and restart VS Code, or use py as the command.
npx not foundInstall Node.js LTS and restart the terminal.
Server disconnects immediatelyRun python server.py directly to reveal import or syntax errors.
Changes do not appearDisconnect and reconnect so Inspector starts a fresh server process.

Cleanup

Press Ctrl+C to stop Inspector, then run deactivate.

Summary
You used Inspector to test discovery, schemas, tool calls, resources, prompts, errors, and the MCP connection lifecycle without building an LLM agent.
