Simple RAG Demo

Build an end-to-end Retrieval-Augmented Generation system in Google Colab using GitHub issues, BGE embeddings, FAISS, a 4-bit Zephyr model, and LangChain.

Intermediate90-120 minutesGoogle Colab GPU
Based on: Hugging Face Open-Source AI Cookbook, Simple RAG for GitHub issues using Hugging Face Zephyr and LangChain.

Learning objectives

  • Explain why RAG can add current, domain-specific knowledge without fine-tuning an LLM.
  • Load GitHub issues as LangChain documents and inspect their metadata.
  • Chunk documents, create embeddings, store them in FAISS, and retrieve relevant chunks.
  • Load Zephyr-7B with 4-bit quantization on a Colab GPU.
  • Construct a prompt, LLM chain, and RAG chain with LangChain Expression Language.
  • Compare the same model with and without retrieved context and inspect its sources.

RAG architecture

GitHub issues
Text chunks
Embeddings
FAISS search
Zephyr prompt
Answer + sources

RAG does not update Zephyr's weights. It retrieves relevant external text and includes it in the model prompt at inference time.

Prerequisites

  • A Google account with access to Google Colab and preferably a T4 GPU.
  • A GitHub account and a personal access token that can read public repositories.
  • The supplied simple_rag_demo.ipynb notebook.
Resource requirement: Zephyr has about 7 billion parameters. This lab requires 4-bit quantization and a GPU with roughly 15 GB memory. A free Colab T4 normally works, but GPU availability is not guaranteed.

4. Load all PEFT GitHub issues

Colab cell - GitHubIssuesLoader
from langchain_community.document_loaders import GitHubIssuesLoader

loader = GitHubIssuesLoader(
    repo="huggingface/peft",
    access_token=ACCESS_TOKEN,
    include_prs=False,
    state="all",
)

docs = loader.load()
print("Issues loaded:", len(docs))
  • repo identifies the owner and repository.
  • include_prs=False excludes pull requests, which GitHub also represents through the Issues API.
  • state="all" loads both open and closed issues.

Loading and pagination may take several minutes. Inspect one resulting Document:

Colab cell - Inspect a document
first_doc = docs[0]
print("Metadata keys:", first_doc.metadata.keys())
print("Metadata:", first_doc.metadata)
print("Content preview:\n", first_doc.page_content[:1000])
Document anatomy: page_content contains text to embed. metadata retains fields such as issue URL, title, labels, state, or creator for tracing sources.

5. Split issues into chunks

Colab cell - Recursive splitter
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=512,
    chunk_overlap=30,
)

chunked_docs = splitter.split_documents(docs)
print("Original issues:", len(docs))
print("Chunks created:", len(chunked_docs))
print("First chunk length:", len(chunked_docs[0].page_content))
print("First chunk preview:\n", chunked_docs[0].page_content[:600])

RecursiveCharacterTextSplitter tries paragraph, line, word, and character boundaries. The 30-character overlap reduces the chance that a useful idea is cut exactly at a chunk boundary.

Tradeoff: Small chunks improve retrieval precision but may lose surrounding explanation. Large chunks preserve context but can add irrelevant text and consume more prompt tokens.

6. Create BGE embeddings and FAISS

Colab cell - Embeddings and vector store
from langchain_huggingface import HuggingFaceEmbeddings
from langchain_community.vectorstores import FAISS

embedding_model = HuggingFaceEmbeddings(
    model_name="BAAI/bge-base-en-v1.5",
    model_kwargs={"device": "cuda"},
    encode_kwargs={"normalize_embeddings": True},
)

sample_vector = embedding_model.embed_query("How do PEFT adapters work?")
print("Embedding dimensions:", len(sample_vector))

db = FAISS.from_documents(chunked_docs, embedding_model)
print("FAISS vector store created.")
  1. BGE converts every chunk into a fixed-length dense vector.
  2. Normalization makes vector direction appropriate for cosine-like similarity comparisons.
  3. FAISS indexes the vectors and keeps the associated LangChain documents.
Most time-consuming ingestion step: embedding thousands of chunks can take several minutes. Do not rerun it unnecessarily.

7. Configure and test the retriever

Colab cell - Similarity retriever
retriever = db.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 4},
)

question = "How do you combine multiple adapters?"
retrieved_docs = retriever.invoke(question)

print("Retrieved chunks:", len(retrieved_docs))
for index, doc in enumerate(retrieved_docs, start=1):
    print(f"\n--- RESULT {index} ---")
    print("Source:", doc.metadata.get("url", doc.metadata.get("source", "unknown")))
    print(doc.page_content[:700])

The retriever embeds the question with the same BGE model and returns the four closest chunk vectors. Read the text now: generation cannot repair fundamentally irrelevant retrieval.

8. Load quantized Zephyr-7B

Colab cell - 4-bit model
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig

model_name = "HuggingFaceH4/zephyr-7b-beta"

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_use_double_quant=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.float16,
)

tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    quantization_config=bnb_config,
    device_map="auto",
)
model.eval()
print("Model device:", model.device)
SettingMeaning
load_in_4bitStores most weights using four bits to reduce memory.
use_double_quantQuantizes quantization constants for additional savings.
nf4A 4-bit format designed for normally distributed neural-network weights.
float16Computation type compatible with the common Colab T4. The source recipe used bfloat16.
device_map="auto"Lets Accelerate place model modules on available hardware.

9. Create the text-generation LLM

Colab cell - Generation pipeline
from transformers import pipeline
from langchain_huggingface.llms import HuggingFacePipeline

text_generation_pipeline = pipeline(
    task="text-generation",
    model=model,
    tokenizer=tokenizer,
    temperature=0.2,
    do_sample=True,
    repetition_penalty=1.1,
    return_full_text=False,
    max_new_tokens=300,
)

llm = HuggingFacePipeline(pipeline=text_generation_pipeline)
print("LangChain LLM is ready.")
  • Low temperature limits randomness, although do_sample=True means exact answers can still vary.
  • Repetition penalty discourages loops.
  • return_full_text=False returns only the generated answer, not the entire prompt.
  • HuggingFacePipeline exposes the Transformers pipeline as a LangChain Runnable.

10. Define the Zephyr prompt and LLM chain

Colab cell - Prompt and LCEL chain
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import PromptTemplate

prompt_template = """<|system|>
Answer the question using the supplied context. If the context does not contain
the answer, say that you do not know. Do not invent repository-specific facts.

Context:
{context}
</s>
<|user|>
{question}
</s>
<|assistant|>
"""

prompt = PromptTemplate(
    input_variables=["context", "question"],
    template=prompt_template,
)

llm_chain = prompt | llm | StrOutputParser()
print(prompt.format(context="EXAMPLE CONTEXT", question="EXAMPLE QUESTION"))

The control tokens follow Zephyr's chat format. The LCEL pipe operator sends the formatted prompt into the LLM and parses its output as a string.

11. Connect retrieval to generation

Colab cell - RAG chain
from langchain_core.runnables import RunnableLambda, RunnablePassthrough

def format_docs(documents):
    return "\n\n".join(
        f"Source {index}:\n{doc.page_content}"
        for index, doc in enumerate(documents, start=1)
    )

rag_chain = (
    {
        "context": retriever | RunnableLambda(format_docs),
        "question": RunnablePassthrough(),
    }
    | llm_chain
)

print("RAG chain is ready.")
  1. The input question goes to the retriever.
  2. The retrieved Document objects are converted into one context string.
  3. RunnablePassthrough preserves the original question.
  4. The context and question fill the prompt template.
  5. Zephyr generates the final answer.

13. Preserve source documents

Colab cell - Answer with sources
def answer_with_sources(user_question):
    source_docs = retriever.invoke(user_question)
    context = format_docs(source_docs)
    answer = llm_chain.invoke({"context": context, "question": user_question})
    return answer, source_docs

answer, sources = answer_with_sources(question)
print("ANSWER\n", answer)

print("\nSOURCES")
for index, doc in enumerate(sources, start=1):
    print(index, doc.metadata.get("url", doc.metadata.get("source", "unknown")))
Evaluation habit: read the source chunks and URLs. A fluent answer is not evidence that retrieval was relevant or that the generated statements are supported.

Important limitations

  • GitHub issues can contain outdated, incomplete, or incorrect user reports.
  • Similarity does not guarantee relevance, and retrieval does not guarantee factual generation.
  • This notebook has no reranker, citations inside generated sentences, evaluation framework, or prompt-injection protection.
  • Updating issues requires rebuilding or incrementally updating the vector store.
  • The notebook stores FAISS only in Colab memory; the index disappears when the runtime is deleted.

Troubleshooting

ProblemSuggested action
No GPU or CUDA errorSelect a T4 GPU and reconnect. Run !nvidia-smi before model loading.
GitHub 401/403Generate a valid token, verify read access and expiration, and rerun only the secret and loader cells.
GitHub rate limitWait, use an authenticated token, and avoid repeatedly loading all issues.
CUDA out of memoryRestart the runtime, ensure 4-bit loading is enabled, and do not load the model twice.
bitsandbytes errorConfirm the runtime has an NVIDIA GPU, restart after installation if necessary, then rerun from the top.
LangChain import errorRerun the installation cell; current integrations are split across community, text-splitter, and Hugging Face packages.
Poor RAG answerInspect retrieval first. Adjust question wording, chunking, k, corpus quality, or add documentation to the corpus.
