1Defining and Calling Functions

A function is a named block of reusable code. Instead of copying the same lines in multiple places, you define the logic once and call it as many times as you need.

1a — Why functions? The problem with repeated code

print_time_repeated_code.py — Before functions
import datetime

# Task 1
print('task completed')
print(datetime.datetime.now())
print()

# Task 2 — copy-pasted!
print('task completed')
print(datetime.datetime.now())
print()

# If you need to change the message, you must find and update EVERY copy

1b — Refactoring with a function

print_time_function.py
import datetime

# Define the function once
def print_time():
    print('task completed')
    print(datetime.datetime.now())
    print()

# Call it as many times as needed — change once, updates everywhere
print_time()

for x in range(0, 10):
    print(x)

print_time()
Rule of thumb (DRY): "Don't Repeat Yourself." If you write the same code more than twice, put it in a function.

1c — Getting initials — building towards a function

get_initials.py — Without a function
# Ask for a name and return the initials

2Function Parameters & Return Values

Parameters let you pass data into a function. Return values let the function send data back to the caller.

2a — A function that takes a parameter and returns a value

get_initials_function.py
# Parameters:
#   name: the full name string to extract the initial from
# Return value:
#   the first letter of the name, uppercased
def get_initial(name):
    initial = name[0:1].upper()
    return initial

# Ask for a name and call the function
first_name = input('Enter your first name: ')
first_name_initial = get_initial(first_name)

print('Your initial is: ' + first_name_initial)

2b — Multiple parameters

get_initials_multiple_parameters.py
def get_initial(name, force_uppercase=True):
    """Return the first letter of name.
       If force_uppercase is True (the default), return it in uppercase.
    """
    if force_uppercase:
        return name[0:1].upper()
    else:
        return name[0:1]

# Using the default (uppercase = True)
print(get_initial('susan'))      # S

# Overriding the default
print(get_initial('susan', force_uppercase=False))   # s
Concept Syntax What it does
Parameter def fn(param): Declares a variable that receives the caller's value
Argument fn('value') The actual value passed when calling the function
Return value return result Sends a value back to wherever the function was called
Default value def fn(x=True): Used when the caller does not supply that argument
Named argument fn(x=False) Makes call sites self-documenting

3Using Packages

Python's standard library includes hundreds of modules. Third-party packages (installed with pip) extend Python's capabilities even further.

3a — Importing a built-in module

Import styles
# Style 1: import the whole module — use datetime.datetime.now()
import datetime
print(datetime.datetime.now())

# Style 2: import just the class you need — use datetime.now() directly
from datetime import datetime
print(datetime.now())

3b — Installing and importing a third-party package

Terminal — Install requests
pip install requests
Use the installed package
import requests

response = requests.get('https://httpbin.org/get')
print(response.status_code)   # 200 means success
print(response.text)          # raw response body
Virtual Environments (best practice): Before installing packages for a project, create a virtual environment with python -m venv venv and activate it with venv\Scripts\activate (Windows) or source venv/bin/activate (Mac/Linux). This keeps project dependencies isolated.

4Calling a REST API

A REST API is a web service you communicate with over HTTP. You send a request (GET, POST, etc.) and receive a response, usually formatted as JSON.

4a — Full API call example (Microsoft Computer Vision)

call_api.py
import requests
import json

# Replace these with your own Azure Computer Vision credentials
SUBSCRIPTION_KEY = "YOUR_KEY_HERE"
vision_service_address = "https://canadacentral.api.cognitive.microsoft.com/vision/v2.0/"

# Build the endpoint URL
address = vision_service_address + "analyze"

# Optional query parameters
parameters = {
    'visualFeatures': 'Description,Color',
    'language': 'en'
}

# Open a local image file in binary mode
image_path = "./TestImages/PolarBear.jpg"
image_data = open(image_path, "rb").read()

# HTTP headers — tell the API what we're sending and who we are
headers = {
    'Content-Type': 'application/octet-stream',
    'Ocp-Apim-Subscription-Key': SUBSCRIPTION_KEY
}

# Make the POST request
response = requests.post(address, headers=headers, params=parameters, data=image_data)

# Raise an exception if the API returned an error code
response.raise_for_status()

# Parse and display the JSON result
results = response.json()
print(json.dumps(results, indent=2))
API Key Security: Never hard-code API keys directly in source files that you commit to version control. Store them in environment variables and read them with os.environ['MY_KEY'].

4b — Understanding the HTTP response

Status Code Meaning
200 OK — request succeeded
400 Bad Request — your request was malformed
401 Unauthorized — invalid or missing API key
404 Not Found — the endpoint URL is wrong
500 Server Error — problem on the API provider's side

5Decorators

A decorator is a function that wraps another function, adding behaviour before or after it runs — without modifying the original function's source code. They are used extensively in frameworks like Flask and Django.

5a — Structure of a decorator

Decorator anatomy
import functools

def my_decorator(func):
    @functools.wraps(func)      # Preserves the original function's metadata
    def wrapper(*args, **kwargs):
        print("--- before ---")
        result = func(*args, **kwargs)  # Call the original function
        print("--- after ---")
        return result
    return wrapper

@my_decorator                   # Apply the decorator with @ syntax
def greet():
    print("Hello!")

greet()
# Output:
# --- before ---
# Hello!
# --- after ---

5b — Decorator with a parameter (factory pattern)

creating_decorators.py
import functools
from colorama import init, Fore
init()   # Initialise colorama for Windows terminal color support

def color(color_code):
    """Decorator factory — takes a color and returns a decorator."""
    def wrapper(func):
        @functools.wraps(func)
        def runner(*args, **kwargs):
            print(color_code + 'Switching to color mode')
            func(*args, **kwargs)
        return runner
    return wrapper

@color(color=Fore.BLUE)         # Apply with the BLUE color code
def greeter():
    print('Hello, world!!')
    print('Just saying hi again')

greeter()
Install colorama first:
Terminal
pip install colorama
Layer Purpose
Outer function color(color_code) The factory — accepts configuration (the color) and returns a decorator
Middle function wrapper(func) The decorator — receives the function being decorated
Inner function runner(*args, **kwargs) The replacement — runs extra logic, then calls the original function
@functools.wraps(func) Copies the original function's name and docstring onto runner
