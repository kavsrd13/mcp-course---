Control Flow & Collections

Learn how to make decisions in your code with if/elif/else, store multiple values in lists and dictionaries, and repeat actions using for and while loops.

Beginner–Intermediate 45–60 minutes Python 3 + VS Code / Terminal

Learning Objectives

  • Write if, if/else, and if/elif/else blocks to branch program flow.
  • Combine multiple conditions using and, or, and in.
  • Distinguish between lists, ranges, and dictionaries and know when to use each.
  • Add items to lists with .append() and access dictionary values by key.
  • Iterate over collections using for loops.
  • Control repetition with while loops and understand how to avoid infinite loops.

The Complete Flow

Get input
Evaluate condition
Branch (if/elif/else)
Store in collection
Loop through items
Output results

Control flow is what gives your program intelligence. Instead of running every line every time, you can skip, repeat, or branch depending on data.

Prerequisites

  • Completion of Exercise 1: Python Fundamentals & Data Types (or equivalent experience).
  • Python 3.x installed and a working code editor.
  • Familiarity with print(), input(), variables, and basic data types.

1Handling Conditions with if / else

An if statement tests a condition. If the condition is True, the indented block runs. Python uses indentation (4 spaces or 1 tab) to define code blocks — there are no curly braces.

1a — Basic if with a tax check

check_tax.py
# Calculate the tax
# Anything purchased for more than $1.00 is charged a 7% tax
price = input('How much did you pay? ')

# Convert the string to a float (decimal number)
price = float(price)

# Check if the price is greater than or equal to $1.00
if price >= 1.00:
    # Everything over $1.00 is charged 7% tax
    tax = 0.07
    print('Tax rate is: ' + str(tax))

1b — Adding an else clause

add_else.py
price = input('How much did you pay? ')
price = float(price)

if price >= 1.00:
    # Anything $1.00 or more is charged 7% tax
    tax = 0.07
    print('Tax rate is: ' + str(tax))
else:
    # Anything under $1.00 is tax-free
    tax = 0
    print('Tax rate is: ' + str(tax))
Indentation matters! All lines inside an if or else block must be indented by the same amount. A mismatch causes an IndentationError.
Operator Meaning Example
== Equal to price == 1.00
!= Not equal to price != 0
> Greater than price > 1.00
>= Greater than or equal to price >= 1.00
< Less than price < 1.00
<= Less than or equal to price <= 1.00

2Multiple Conditions with elif

When you have more than two possible branches, use elif (short for "else if"). Python checks each condition in order and runs the first block that matches.

2a — Using elif for province tax rates

use_elif.py
province = input('What province do you live in? ')
tax = 0

if province == 'Alberta':
    tax = 0.05
elif province == 'Nunavut':
    tax = 0.05
elif province == 'Ontario':
    tax = 0.13

print(tax)

2b — Using or to combine conditions

or_statements.py
province = input('What province do you live in? ')
tax = 0

# Use 'or' to check multiple values in one condition
if province == 'Alberta' or province == 'Nunavut':
    tax = 0.05
elif province == 'Ontario':
    tax = 0.13

print(tax)

2c — Using in for cleaner multi-value checks

use_in_statements.py
province = input('What province do you live in? ')
tax = 0

# 'in' checks if a value is present in a list — much cleaner!
if province in ['Alberta', 'Nunavut', 'Northwest Territories']:
    tax = 0.05
elif province == 'Ontario':
    tax = 0.13

print(tax)
Best practice: Prefer in [...] over chaining multiple or conditions — it is easier to read and add new values to.

3Collections — Lists & Dictionaries

Collections let you store multiple values in a single variable. Python has four built-in collection types; we focus on the two most common: lists and dictionaries.

3a — Lists: ordered, changeable sequences

lists.py
# Create a list with initial values
names = ['Christopher', 'Susan']

# Create an empty list and add items with .append()
scores = []
scores.append(98)
scores.append(99)

print(names)   # ['Christopher', 'Susan']
print(scores)  # [98, 99]

3b — Dictionaries: key-value pairs

dictionaries.py
# Create a dictionary with one key-value pair
person = {'first': 'Christopher'}

# Add a new key-value pair using square brackets
person['last'] = 'Harrison'

print(person)           # {'first': 'Christopher', 'last': 'Harrison'}
print(person['first'])  # Christopher
Feature List Dictionary
Syntax [item1, item2] {'key': 'value'}
Access by Index (0, 1, 2…) Key (any string or number)
Ordered Yes Yes (Python 3.7+)
Best for Ordered sequences of items Named properties of a single object

4for Loops

A for loop iterates over every item in a collection (list, string, range, etc.) and runs the indented block once per item.

4a — Iterating over a list

for.py
for name in ['Christopher', 'Susan']:
    print(name)

# Output:
# Christopher
# Susan

4b — Looping over a range of numbers

number.py
# range(start, stop) generates numbers from start up to (but not including) stop
for x in range(0, 5):
    print(x)

# Output: 0, 1, 2, 3, 4
Remember: range(0, 5) produces 0, 1, 2, 3, 4 — it stops before the end value. This matches how Python list indices work (0-based).

5while Loops

A while loop keeps running as long as its condition is True. You must update the condition variable inside the loop or you will create an infinite loop.

5a — while loop with a counter

while.py
names = ['Christopher', 'Susan']
index = 0

while index < len(names):      # len() returns the number of items in the list
    print(names[index])
    index = index + 1           # CRITICAL: increment index to avoid infinite loop

# Output:
# Christopher
# Susan
Danger — Infinite Loops: If you forget to increment index (or change the condition variable), the loop will run forever. Use Ctrl+C in the terminal to stop a running program.
Loop Type Best used when Example
for You know the collection or number of iterations in advance Print each name in a list
while You loop until a condition changes (unknown number of iterations) Keep asking for input until the user types 'quit'

🏆 Coding Challenge

Build a mini grade-book program:

  1. Create an empty list called grades.
  2. Use a while loop to ask the user to enter a grade (number). If they type done, stop the loop; otherwise .append() the grade (as a float) to the list.
  3. After the loop, use a for loop to print each grade.
  4. Calculate and print the average of all grades.
  5. Use an if/elif/else block to print a letter grade: A (≥90), B (≥80), C (≥70), D (≥60), F (below 60).
Hints: Use sum(grades) / len(grades) to calculate the average. Wrap the float conversion in a try/except to handle non-numeric input.

Summary

In this exercise you added decision-making and repetition to your Python toolkit:

  • if / else — branch based on a single condition
  • elif — handle multiple mutually exclusive conditions
  • and / or / in — combine or simplify conditions
  • Lists — ordered, mutable sequences; add items with .append()
  • Dictionaries — key-value stores for named properties
  • for loops — iterate a fixed number of times or over a collection
  • while loops — repeat until a condition becomes False
