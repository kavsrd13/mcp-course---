Build your first MCP server

Create a small Python server that teaches the difference between MCP tools, resources, and prompts.

Beginner30 minutesPython + VS Code

Learning objectives

After completing this exercise, you will be able to:

  • Create and run an MCP server with FastMCP.
  • Expose actions as tools, data as resources, and reusable instructions as prompts.
  • Use MCP Inspector to discover and invoke the server capabilities.

Prerequisites

  • Python 3.10 or later.
  • Visual Studio Code.
  • Node.js, used to launch MCP Inspector.
Tip: In VS Code, open Terminal > New Terminal. All commands below use PowerShell.

1. Create the project and virtual environment

  1. Create a folder for the exercise and open it in VS Code.
  2. In the VS Code terminal, run:
PowerShell
mkdir mcp-server-basics
cd mcp-server-basics
code .
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install "mcp[cli]"
If activation is blocked: run Set-ExecutionPolicy -Scope Process Bypass, and then activate the environment again. This changes policy only for the current terminal.

When the environment is active, the terminal prompt begins with (.venv).

2. Create the MCP server

  1. In Explorer, create a file named server.py.
  2. Paste the following code into the file and save it.
server.py
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("classroom-basics")

COURSE_NOTES = {
    "tools": "Tools perform actions. The model decides when to call them.",
    "resources": "Resources expose data that a client can read as context.",
    "prompts": "Prompts are reusable message templates supplied by the server.",
}


@mcp.tool()
def add(a: int, b: int) -> int:
    """Add two numbers."""
    return a + b


@mcp.tool()
def attendance_message(student: str, present: bool = True) -> str:
    """Create a simple attendance message."""
    status = "present" if present else "absent"
    return f"{student} is marked {status}."


@mcp.resource("lesson://{topic}")
def lesson(topic: str) -> str:
    """Read a short lesson about an MCP primitive."""
    return COURSE_NOTES.get(topic, f"No lesson found for '{topic}'.")


@mcp.prompt()
def explain_for_beginner(topic: str) -> str:
    """Create a reusable teaching prompt."""
    return (
        f"Explain {topic} to a beginner. Use one analogy and one tiny example. "
        "Keep the answer under 150 words."
    )


if __name__ == "__main__":
    mcp.run()

Understand the code

  • FastMCP creates the server and generates MCP schemas from Python type hints.
  • @mcp.tool() exposes a callable action. Its docstring becomes the tool description.
  • @mcp.resource() exposes readable information at an MCP URI.
  • @mcp.prompt() exposes a reusable message template. It does not call a model by itself.
  • mcp.run() uses the stdio transport, so an MCP client starts and communicates with the process.

3. Inspect the server

Run MCP Inspector from the same folder:

PowerShell
npx @modelcontextprotocol/inspector python server.py
  1. Allow npm to install the Inspector package if prompted.
  2. Open the local Inspector URL printed in the terminal.
  3. Select Connect. The transport should be STDIO, the command python, and the argument server.py.
  4. Open Tools, select add, enter 7 and 5, and run it. The result should be 12.
  5. Call attendance_message with student Asha and present true.
  6. Open Resources and read lesson://tools. Repeat with resources and prompts.
  7. Open Prompts, choose explain_for_beginner, and enter MCP tools.
Expected result: Inspector lists two tools, one resource template, and one prompt. The prompt result is a message template; no LLM response is generated.

4. Make a small change

Add the following tool above the final if block, save the file, and reconnect Inspector:

server.py addition
@mcp.tool()
def multiply(a: int, b: int) -> int:
    """Multiply two numbers."""
    return a * b

Confirm that Inspector now discovers multiply. This demonstrates that the client learns server capabilities during initialization.

Cleanup

  1. Press Ctrl+C in the terminal to stop Inspector.
  2. Run deactivate to leave the virtual environment.
Summary
You built one MCP server with tools, resources, and prompts, then used Inspector as an MCP client to discover and invoke them.
