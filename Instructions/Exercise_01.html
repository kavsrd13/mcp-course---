1Hello World & the Print Statement

The print() function is the simplest way to display information. Every Python program starts here.

1a — Your first line of Python

hello_world.py
# the print statement displays a message
print('Hello world')

Run it: Save the file and run python hello_world.py. You should see Hello world in the terminal.

1b — Blank lines and newline characters

print_blank_line.py
# Each print statement starts on a new line
print('Hello world')

# Passing nothing to print() produces a blank line
print()
print('Did you see that blank line?')

# '\n' is a special character that means "new line"
print('Blank line \nin the middle of string')

1c — Asking the user for input

ask_for_input.py
# The input() function prompts the user for a value
# Store the result in a variable
name = input('What is your name? ')

print(name)
Try it: Run the script, type your name, and press Enter. The program prints back what you typed. Notice that input() always returns a string.

2Comments & Code Documentation

Comments are lines that Python ignores completely. They exist only for the reader. Good comments explain why something is done, not what is done (the code already shows what).

2a — Comments are not executed

comments_are_not_executed.py
# This is a comment - Python will not run this line
print('This line will run')

# You can comment out code to temporarily disable it
# print('This line is commented out and will NOT run')

2b — Using comments for debugging

comments_for_debugging.py
# Comment out lines to isolate bugs
# print('debug: checking value here')
print('Normal program output')
Note: Python does not have a built-in multi-line comment syntax. To comment multiple lines, prefix each line with #, or use a triple-quoted string """...""" as a workaround.

3String Variables & Methods

Strings store text. Python provides a rich set of built-in methods to manipulate them.

3a — Storing strings in variables

strings_in_variables.py
# You can store strings in variables
first_name = 'Susan'

# The variable can then be used later in your code
print(first_name)

3b — Formatting strings with user input

format_strings.py
# Ask the user for their first and last name
first_name = input('What is your first name? ')
last_name  = input('What is your last name? ')

# capitalize() makes the first letter uppercase, rest lowercase
print('Hello ' + first_name.capitalize() + ' ' + last_name.capitalize())

3c — Common string methods

string_functions.py
sentence = 'The dog is named Sammy'

print(sentence.upper())       # ALL UPPERCASE
print(sentence.lower())       # all lowercase
print(sentence.capitalize())  # First letter uppercase, rest lowercase
print(sentence.count('a'))    # Count occurrences of 'a' → 2
Method What it does Example output
.upper() Returns the string in all uppercase letters THE DOG IS NAMED SAMMY
.lower() Returns the string in all lowercase letters the dog is named sammy
.capitalize() First letter uppercase, rest lowercase The dog is named sammy
.count(x) Counts how many times x appears 2

4Numeric Variables & Math

Python distinguishes between integers (int) and floating-point numbers (float). You cannot directly add a number to a string — you must convert first.

4a — Performing arithmetic

doing_math.py
first_num  = 6
second_num = 2

print('addition:')       ; print(first_num + second_num)   # 8
print('subtraction:')    ; print(first_num - second_num)   # 4
print('multiplication:') ; print(first_num * second_num)   # 12
print('division:')       ; print(first_num / second_num)   # 3.0
print('exponent:')       ; print(first_num ** second_num)  # 36

4b — Mixing strings and numbers

combining_strings_and_numbers.py
days_in_feb = 28

print(days_in_feb)   # works fine — prints the number

# This will cause a TypeError — cannot add int and str
# print(days_in_feb + ' days in February')

# Convert the number to a string first
print(str(days_in_feb) + ' days in February')   # 28 days in February
Key rule: Use str() to convert a number to a string before concatenating with +. Alternatively, use an f-string: f"{days_in_feb} days in February".

5Dates & Times

Python's built-in datetime module lets you get the current date, extract parts of it, and format it for display.

5a — Getting the current date and time

get_current_date.py
from datetime import datetime

current_date = datetime.now()
# datetime.now() returns a datetime object — convert to string to concatenate
print('Today is: ' + str(current_date))

5b — Accessing individual date components

format_date.py
from datetime import datetime

today = datetime.now()

# Access individual components — all return integers
print('Day:    ' + str(today.day))
print('Month:  ' + str(today.month))
print('Year:   ' + str(today.year))
print('Hour:   ' + str(today.hour))
print('Minute: ' + str(today.minute))
print('Second: ' + str(today.second))

6Error Handling

Python errors fall into three categories. Understanding them helps you write robust code.

Error Type When it occurs Example
Syntax Error Before the program runs — Python cannot parse the code print('hello' (missing closing paren)
Runtime Error While the program is running — something unexpected happens Dividing by zero
Logic Error Program runs but produces the wrong result Checking x < y when you meant x > y

6a — Catching a runtime error with try/except

runtime.py
x = 42
y = 0

try:
    print(x / y)                        # This will raise ZeroDivisionError
except ZeroDivisionError as e:
    print('Sorry, something went wrong') # Specific handler
except:
    print('Something really went wrong') # Catch-all handler
finally:
    print('This always runs on success or failure')

6b — Logic error example

logic.py
x = 206
y = 42

# BUG: The condition is backwards — x IS greater than y,
# but the message says it is, yet the condition checks x < y
if x < y:
    print(str(x) + ' is greater than ' + str(y))

# FIX: Change x < y to x > y
if x > y:
    print(str(x) + ' is greater than ' + str(y))
Logic errors are the hardest to find because Python does not report them — the program runs without crashing but produces incorrect results. Always test with known values.
