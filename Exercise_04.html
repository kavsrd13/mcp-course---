Create a generative AI chat app

In this exercise, you use the OpenAI SDK to build chat apps with both the Chat Completions API and the Responses API against a model deployed in a Microsoft Foundry project.

This exercise takes approximately 45 minutes.

Note: Some of the technologies used in this exercise are in preview or in active development. You may experience some unexpected behavior, warnings, or errors.

Learning objective

In this lab, you will learn:

  • How to call the Chat Completions API for message-based chat prompts.
  • How to call the Responses API for instruction/input style requests with response tracking.
  • The practical differences between these two API styles and when to use each.

Prerequisites

Before starting this exercise, ensure you have:

  • An active Azure subscription
  • Visual Studio Code installed
  • Python version 3.13.xx installed*
  • Azure CLI installed

* Python 3.14 is available, but some dependencies are not yet compiled for that release. The lab has been successfully tested with Python 3.13.12.

Create a Microsoft Foundry project

Microsoft Foundry uses projects to organize models, resources, data, and other assets used to develop an AI solution.

  1. In a web browser, open the Microsoft Foundry portal at https://ai.azure.com and sign in using your Azure credentials. Close any tips or quick start panes that are opened the first time you sign in, and if necessary use the Foundry logo at the top left to navigate to the home page.

  2. If it is not already enabled, in the tool bar the top of the page, enable the New Foundry option. Then, if prompted, create a new project with a unique name; expanding the Advanced options area to specify the following settings for your project:
    • Foundry resource: Use the default name for your resource (usually {project_name}-resource)
    • Subscription: Your Azure subscription
    • Resource group: Create or select a resource group
    • Region: Select any of the AI Foundry recommended regions in this list
  3. Wait for your project to be created. Then view its home page.

Deploy a model

In this version of the lab, you will create an agent in the New Foundry experience. When the agent is created, a model deployment is created automatically.

  1. In the Foundry portal, ensure New Foundry is enabled.
  2. Open the left navigation and select Agents.
  3. Select + New agent and create an agent with default settings.
  4. Wait until agent creation finishes. A model deployment is created automatically for agent usage.
  5. Go to Models and note the following values:
    • Deployment name (use your own deployment name from the portal)
    • Azure OpenAI endpoint (example format: https://YOUR-RESOURCE.services.ai.azure.com/openai/v1)
    • API key (from your resource keys)

Use the ChatCompletions API to chat with the model

Create a new Python file named chat_app.py. Add the code in parts.

Part 1: Imports and authentication

This part imports the OpenAI SDK and Azure identity helpers. DefaultAzureCredential allows Entra ID sign-in. The token provider gives OpenAI a bearer token for Azure AI.

from openai import OpenAI
from azure.identity import DefaultAzureCredential, get_bearer_token_provider

Part 2: Connection settings

This part sets the endpoint and deployment name. Students can paste direct values first and later switch to environment variable loading if needed.

endpoint = "https://YOUR-RESOURCE.services.ai.azure.com/openai/v1"
deployment_name = "YOUR_DEPLOYMENT_NAME"

Part 3: Token provider and client

This part creates an Entra token provider and initializes the OpenAI client for Azure endpoint access.

token_provider = get_bearer_token_provider(
    DefaultAzureCredential(),
    "https://ai.azure.com/.default"
)

client = OpenAI(
    base_url=endpoint,
    api_key=token_provider
)

Part 4: Send a chat request

This part sends one user message to the deployment and prints the assistant response.

completion = client.chat.completions.create(
    model=deployment_name,
    messages=[
        {
            "role": "user",
            "content": "What is the capital of France?",
        }
    ],
)

print(completion.choices[0].message.content)
  1. Run Azure sign-in in your terminal: 
    az login
  2. Create chat_app.py, combine the code from Parts 1 to 4, and save.
  3. Run the script: 
    python chat_app.py
  4. Expected output: a short answer such as Paris.

Chat Completions vs Responses API

Before writing the second file, review the API differences:

  • Chat Completions: Uses a messages array with explicit roles like system, user, and assistant.
  • Responses: Uses instructions and input, and supports conversation continuation with previous_response_id.
  • Use case: Chat Completions is ideal for explicit message control, while Responses is often simpler for workflow-driven prompt/response handling.

Use the Responses API with response.py

Create a new file named response.py and add the following code in parts.

Part 1: Client configuration

This part sets endpoint, API key, and deployment name for API key authentication.

from openai import OpenAI

# Use values from your Foundry/Azure portal
azure_openai_endpoint = "https://YOUR-RESOURCE.openai.azure.com/openai/v1"
api_key = "YOUR_API_KEY"
model_deployment = "YOUR_DEPLOYMENT_NAME"

openai_client = OpenAI(
    base_url=azure_openai_endpoint,
    api_key=api_key,
    default_headers={"api-key": api_key}
)

Part 2: Build a chat loop

This part creates an interactive loop and tracks last_response_id to keep context between turns.

def chat_with_model():
    print("--- Chatting with model using Responses API (Type 'quit' to exit) ---")
    last_response_id = None

    while True:
        input_text = input("\nYou: ")
        if input_text.lower() in ["quit", "exit"]:
            break

Part 3: Call the Responses API

This part sends the request using instructions, input, and previous_response_id.

        try:
            response = openai_client.responses.create(
                model=model_deployment,
                instructions="You are a helpful AI assistant that answers questions and provides information.",
                input=input_text,
                previous_response_id=last_response_id,
            )

            print(f"Assistant: {response.output_text}")
            last_response_id = response.id

        except Exception as e:
            print(f"Error occurred: {e}")

Part 4: Run the file

if __name__ == "__main__":
    chat_with_model()
  1. Save the file as response.py.
  2. Run it: 
    python response.py
  3. Ask two related questions (for example, first ask about ELIZA, then ask how it compares with modern LLMs) and observe context retention.

Security reminder: Never hardcode real keys in source files you plan to share or commit. Prefer environment variables for production use.

Optional: use values from .env inside Python

If you prefer not to hardcode endpoint and deployment values in source code, use this version:

import os
from dotenv import load_dotenv
from openai import OpenAI
from azure.identity import DefaultAzureCredential, get_bearer_token_provider

load_dotenv()

endpoint = os.environ["AZURE_OPENAI_ENDPOINT"]
deployment_name = os.environ["AZURE_OPENAI_DEPLOYMENT"]
token_provider = get_bearer_token_provider(DefaultAzureCredential(), "https://ai.azure.com/.default")

client = OpenAI(base_url=endpoint, api_key=token_provider)

completion = client.chat.completions.create(
    model=deployment_name,
    messages=[{"role": "user", "content": "What is the capital of France?"}],
)

print(completion.choices[0].message.content)

Summary

In this exercise, you created a Foundry project, used the New Foundry agent flow to get an automatic model deployment, captured deployment settings, and implemented two client styles: one with Chat Completions and one with Responses. You also compared the API patterns and practiced context handling with previous_response_id.

Clean up

If you’ve finished exploring Microsoft Foundry, you should delete the resources you have created in this exercise to avoid incurring unnecessary Azure costs.

  1. Open the Azure portal and view the contents of the resource group where you deployed the resources used in this exercise.
  2. On the toolbar, select Delete resource group.
  3. Enter the resource group name and confirm that you want to delete it.
